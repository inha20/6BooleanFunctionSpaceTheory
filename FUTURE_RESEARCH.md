| 우선순위 | 연구 방향 | 연결 Phase |
|---------|---------|---------|
| 1 | SRT 인지 실험 (H3·H4·H6 실증) | Phase 3 |
| 2 | 인간-AI 협업 모델 및 교육 프레임워크 | Phase 4 |
| 3 | 이론 일반화 (n=5 확장, 다치 논리, 위상학적 관점) | 장기 |
| 4 | 연구 기원 타임라인 완성 | paper.md §6 보완 |

## 2. 인간-AI 협업 모델 및 교육 프레임워크 (Phase 4)

### 2.1 기억 분담 모델 (Memory Division Model)

> **상세 내용:** `theory/MemoryAndEducationModel.md` 참조

장기 인간-AI 협업에서 두 종류의 기억이 상보적으로 작동한다:

$$K(t) = M_H(t) \cup M_{doc}(t)$$

| 기억 주체 | 특성 | 한계 |
|---------|-----|-----|
| $M_H(t)$ (연구자) | 연속적, 맥락 풍부, 직관 포함 | 부정확성, 망각 |
| $M_{doc}(t)$ (문서) | 대용량, 정확, 빠른 검색 | 세션 종료 시 M_AI 소멸 |

**핵심 발견:** SESSION_START.md가 단일 진입점으로 기능하는 것은 이 모델에서 $M_{doc}(t)$의 인터페이스 계층 최적화이다.

**남은 작업:**
- [ ] Level 1~3 교육 시나리오를 실제 학생 대상 파일럿 실습 또는 워크숍으로 실행
- [ ] 협업 수준(Level) 전이 조건 연구

### 2.2 최적 역할 분담 모델 — 탐구 루프

> **상세 내용:** `fq/FQ_E_human_ai.md` §FQ-16 참조

Boolean 함수 공간 탐구에서 인간과 AI의 역할 분담:

```
[인간] 흥미로운 패턴 선별 / 질문 생성
          ↓
[AI] 해당 패턴의 Space(G) 분류 계산 + 유사 패턴 전체 열거
          ↓
[인간] 결과 해석 + 새로운 질문 추출
          ↓
[AI] 이론적 근거 탐색 + 문서화
          ↓
[인간] 연구 방향 결정 + 가설 설정
          ↓ (루프 반복)
```

### 2.3 AI 협업 교육 프레임워크

> **상세 내용:** `theory/MemoryAndEducationModel.md` §2, `fq/FQ_E_human_ai.md` §FQ-17 참조

| Level | AI 인식 방식 | 협업 품질 | 이 연구에서의 예 |
|-------|-----------|---------|--------------|
| Level 1 | 도구/계산기 | 낮음 | Space(G) 크기 계산, K-map 생성 |
| Level 2 | 검증자/교차점검 | 중간 | Burnside 결과 반례 탐색 |
| Level 3 | 공동 탐구자 | 높음 | FQ-1~20 질문 생성 과정 전체 |

**교육 원칙:**
1. **질문 우선(Question-First):** 답을 얻기 위해서가 아니라 더 나은 질문을 하기 위해 AI 활용
2. **검증 책임:** AI 출력의 최종 검증 책임은 인간에게 있음
3. **맥락 관리:** 장기 협업에서 컨텍스트를 유지하는 문서화 훈련 (Work-Then-Record)
4. **방향 결정:** AI가 제안하는 방향 중 어느 것을 선택할지는 인간의 판단

### 2.4 paper.md와의 통합 지점

- 기억 분담 모델 공식화 결과 → paper.md §5.4(H8-H9-H10) 강화 가능
- 교육 프레임워크 확립 → 별도 논문(Repo 5 연장) 또는 paper.md 부록

## 3. 이론 일반화 (장기 연구)

### 3.1 차원 확장: n=5 이상

> **상세 내용:** `fq/FQ_F_generalization.md` §FQ-18 참조

n=4 분류 결과(S₄-궤도)를 더 높은 차원으로 확장한다.

**n=5 기초 스캐폴드 (완료):**
- 5변수 아핀 함수: 64개, S₅-궤도 수 **12개 예측** (이론 계산 완료)
- `scripts/s4_orbit_calculator.py --n5` 옵션으로 기초 검증 구현됨

**변수 수별 함수 수 규모:**

| n | 전체 함수 수 | 단조 함수 수 (Dedekind) | 비고 |
|---|-----------|---------------------|-----|
| 4 | 65,536 | 168 | **본 연구 완결** |
| 5 | 4.3×10⁹ | 7,581 | 전수 K-map 분석 불가, 샘플링 필요 |
| 6 | 1.8×10¹⁹ | 7,828,354 | 이론 분석만 가능 |

### 3.2 다치 논리로의 확장

> **상세 내용:** `questions/OQ5_manyvalued_logic.md` 참조

Boolean(2치) → Ternary(3치) → k치 논리로 Space(G) 이론을 확장한다.

**보존되는 구조:** Space(G) 정의(클론 개념), 함수적 완전성(Rosenberg 정리), 변수 치환 대칭(Sₙ 군)

**달라지는 구조:** K-map 기하학(9×9 격자 등), 아핀 공간(GF(k) 위 선형 대수), 만능 게이트

$$|\mathcal{F}_{n,k}| = k^{k^n}$$

n=2, k=3: 19,683개 — 이미 전수 탐구가 어려운 규모.

### 3.3 위상학적 관점

> **상세 내용:** `fq/FQ_F_generalization.md` §FQ-20 참조

Boolean 함수 공간을 위상 공간으로 보면: 함수 간 거리 = Hamming 거리(진리표 비교 시 다른 비트 수). S₄-궤도는 이 위상에서 연결 성분을 이룬다.

**미해결 질문:** Space(G)의 Hamming 거리 기반 위상적 성질은 무엇인가?

### 3.4 paper.md와의 통합 지점

- n=5 궤도 분류 결과 확보 시 → paper.md §3에 새 절 추가
- 다치 논리·위상 결과 → paper.md §7.3(향후 연구) 업데이트
