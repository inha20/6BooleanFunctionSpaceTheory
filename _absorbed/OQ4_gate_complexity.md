## 2. 게이트 복잡성의 두 가지 정의

### 2.1 정의 A: 게이트 G 자체의 복잡성

게이트 G를 기본 게이트(NAND)로 구현하는 데 필요한 최소 NAND 수.

| 게이트 G | NAND 구현 비용 | 메모 |
|---------|--------------|-----|
| NAND | 1 | 기본 게이트 자체 |
| NOR | 3 | NOT(A NAND B) = 3 NAND |
| NOT | 1 | A NAND A |
| AND | 2 | NOT(A NAND B) = 2 NAND |
| OR | 3 | (NOT A) NAND (NOT B) = 3 NAND |
| XOR | 4 | 표준 NAND-XOR = 4 NAND |
| XNOR | 5 | NOT(XOR) |
| IMPLY (A→B) | 2 | A NAND (A NAND B) |

### 2.2 정의 B: 목표 함수 f를 G만으로 구현하는 최소 G의 수

이 수를 **G-복잡성(G-complexity)** of f 라고 한다:

$$C_G(f) = \min \{ k \mid \text{f를 정확히 k개의 G 게이트로 구현 가능} \}$$

## 3. Space(G) 크기 vs. G의 복잡성

### 3.1 관찰된 패턴

| 게이트 G | Space(G) 크기 (4변수) | NAND 구현 비용 |
|---------|-------------------|--------------|
| NAND | 65,536 (최대) | 1 (최소) |
| NOR | 65,536 (최대) | 3 |
| NOT | ~256 (자기쌍대 클론) | 1 |
| AND | ~168 (단조 부분집합) | 2 |
| OR | ~168 (단조 부분집합) | 3 |
| XOR | 32 (아핀, 최소급) | 4 (최대) |

> **가설 OQ-4 (비공식):** 게이트 G의 복잡성(NAND 비용)이 높을수록 Space(G)가 작은 경향이 있다.  
> (단, NOR는 복잡성 3인데 Space(NOR) = Ω로 반례처럼 보임 → 더 정확한 정의 필요)

### 3.2 NAND vs. NOR 역설

NOR는 NAND보다 구현 비용이 3배(3 NAND 필요)임에도 Space(NOR) = Ω:
- 이유: NOR = NOT(A OR B) = NAND의 쌍대(dual) → 만능성 보존
- 결론: "G의 NAND 복잡성" ≠ "Space(G)의 크기"

더 정확한 관계를 찾기 위해 다른 복잡성 개념이 필요하다.

### 3.3 대안적 복잡성 측도: Post 클래스 위반 수

Post의 완전성 정리에서, G가 만능(Space(G)=Ω)이기 위한 조건:  
G가 5가지 Post 클래스(T₀, T₁, M, L, S) **중 어느 것도 보존하지 않아야 함**

| 게이트 G | 위반하는 Post 클래스 수 | Space(G) |
|---------|-------------------|---------|
| NAND | 5 (모두) | Ω (최대) |
| NOR | 5 (모두) | Ω (최대) |
| XOR | 2 (T₁, M 위반) | L (최소급) |
| AND | 0 (보존) | M의 부분집합 (작음) |
| NOT | 3 (T₀, T₁, M 위반) | S (중간) |

> **개선된 가설:** "Post 클래스 위반 수"가 많을수록 Space(G)가 크다.  
> AND와 XOR은 이 패턴의 반례 → 추가 분석 필요

## 4. G-복잡성과 K-map 구조의 관계 (FQ-10)

### 4.1 목표 함수 f의 G-복잡성과 Space(G) 내 위치

Space(G) = {f | C_G(f) < ∞}  
Space(G)가 크면 → G로 구현 가능한 함수가 많음 → C_G(f) < ∞인 함수가 많음

**층 구조 (Layered Structure):**
Space(G)를 복잡성에 따라 층으로 나눌 수 있다:

```
Layer 0: G 자체 (k=1)
Layer 1: G(G(·), ·), G(·, G(·)) 등 (k=2)
Layer 2: 더 복잡한 구성 (k=3)
...
Layer ∞: Space(G)의 전체
```

각 Layer는 K-map 위에서 어떤 패턴들을 포함하는가?

### 4.2 XOR의 층 구조 분석

**Layer 0 (XOR 자체):**
XOR(A,B): K-map = 수평 줄무늬 (2변수 패리티)

**Layer 1 (XOR 2개):**
XOR(XOR(A,B), C) = A⊕B⊕C: K-map = 3변수 패리티

**Layer 2 (XOR 3개):**
XOR(A⊕B⊕C, D) = A⊕B⊕C⊕D: K-map = 완전 체커보드

→ XOR의 K-map 패턴이 층이 올라갈수록 "더 복잡한 체커보드 구조"로 발전

### 4.3 NAND의 층 구조 분석

**Layer 0:** NAND(A,B) — 1개의 0만 있는 K-map  
**Layer 1:** NOT(A) = NAND(A,A), AND(A,B) = NOT(NAND(A,B))  
**Layer 2:** OR(A,B), XOR(A,B)  
**Layer 3:** 더 복잡한 함수들  
...  
**궁극:** 65,536개 전체 = Space(NAND) = Ω

## 5. 복잡성과 K-map 패턴의 정성적 관계

| G-복잡성 수준 | K-map 패턴 특성 |
|------------|--------------|
| 낮음 (k=1,2) | 단순한 패턴 (줄무늬, 단일 0/1 영역) |
| 중간 (k=3~5) | 규칙적이지만 복잡한 패턴 |
| 높음 (k≥6) | 비규칙적, 임의적 패턴 |
| 무한 (∉ Space(G)) | G로는 구현 불가 |
