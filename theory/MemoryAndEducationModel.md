## 0. 이 문서의 위치와 목적

이 문서는 **장기 인간-AI 공동 연구에서 기억이 어떻게 분담되는가**를 수학적·인지적으로 모델링하고, 그 모델을 AI 협업 교육에 적용하기 위한 교육 프레임워크를 제안한다.

Structure Recognition Research Program의 가장 두드러진 특징 중 하나는 **수십 번의 AI 세션에 걸쳐 일관성 있는 연구 진행이 가능했다**는 점이다. 이것은 우연이 아니다. 이 문서는 그 가능성의 인지적·수학적 기반을 탐구한다.

## 1. 기억 분담 모델 (Memory Division Model)

### 1.1 핵심 관찰

장기 인간-AI 협업에서 두 종류의 기억이 작용한다:

| 기억 주체 | 특성 | 한계 |
|---------|-----|-----|
| **M_AI(t)** — 세션 내 AI 기억 | 대용량, 정확, 빠른 검색 | 세션 종료 시 초기화; 연속성 없음 |
| **M_H(t)** — 연구자의 장기 기억 | 연속적, 맥락 풍부, 감정·직관 포함 | 부정확성, 재구성 오류, 망각 |

이 두 기억 체계는 **상보적 결함(complementary deficits)**을 가진다: AI는 연속성을 갖지 못하고, 인간은 정확성과 대용량 검색에 한계가 있다.

### 1.2 수학적 모델

세션 t에서 협업의 실제 접근 가능한 지식 상태를 K(t)라 하면:

```
K(t) = M_H(t) ∪ M_doc(t)
```

여기서:
- **M_H(t):** 연구자가 세션 t 시작 시점에 가지고 있는 장기 기억
- **M_doc(t):** 세션 t에서 AI가 접근할 수 있는 외재화된 문서 기억 (SESSION_START.md, 이론 문서, 코드 등)

AI의 세션 내 기억 M_AI(t)는 세션이 시작되면:

```
M_AI(t) = Process(M_doc(t), Query(t))
```

즉 AI는 M_doc(t)을 입력으로 받아 처리하며, 세션 내 생성 지식을 축적한다. 세션이 끝나면 M_AI(t)는 소멸하지만, 세션 중에 생성된 새 문서들이 M_doc(t+1)으로 편입된다:

```
M_doc(t+1) = M_doc(t) ∪ Output(M_AI(t))
```

이 재귀적 구조가 **세션 간 연속성의 수학적 기반**이다.

### 1.3 시스템의 핵심 취약점

위 모델에서 시스템의 취약점은 명확히 드러난다:

**허위 완료 기록(False Completion Records):** M_doc(t)에 실제 완료되지 않은 항목이 완료로 기록되면, M_AI(t)는 오염된 상태 정보를 기반으로 작업한다. 이것이 Session 45에서 scripts/ 폴더 문제와 Session 46에서 MemoryAndEducationModel.md·ResearchOriginTimeline.md 누락이 발생한 원인이다.

**Work-Then-Record Protocol은 이 취약점에 대한 직접적 대응이다:**
```
실제 파일 생성·실행·검증 → 파일 re-read → 현황판 기록
```

### 1.4 SESSION_START.md의 역할

SESSION_START.md는 이 모델에서 **M_doc(t)의 인터페이스 계층**을 담당한다:

| SESSION_START.md 섹션 | 모델에서의 역할 |
|--------------------|------------|
| Active Queue | K(t)에서 아직 M_AI(t)로 이전되지 않은 작업 상태 |
| Repository Health | M_doc(t)의 전체 상태 요약 (빠른 접근용) |
| AI Operating Principles | M_AI(t)가 M_doc(t)를 처리하는 방식을 제약하는 메타-규칙 |
| Four-Paper Architecture | M_H(t)의 연구 구조적 맥락을 M_doc(t)로 외재화한 것 |
| Concept Genealogy | M_H(t)의 역사적 발전 경로를 M_doc(t)로 외재화한 것 |

SESSION_START.md가 단일 진입점으로 설계된 이유는 M_AI(t) 초기화 비용을 최소화하기 위해서다. AI는 세션 시작 시 이 파일 하나만 읽고도 M_doc(t)의 핵심 상태에 접근할 수 있다.

## 2. AI 협업 교육 프레임워크 (Education Framework)

### 2.1 왜 AI 협업 교육이 필요한가

AI 협업의 질은 사용자가 AI를 **어떤 인지적 역할로 인식하는가**에 따라 크게 달라진다. 다음 세 가지 수준(Level)은 인식의 깊이를 나타낸다:

| Level | AI 인식 방식 | 전형적 사용 패턴 | 협업 품질 |
|-------|-----------|-------------|---------|
| Level 1 | 도구/계산기 | 단순 질의 → 정답 수령 | 낮음 |
| Level 2 | 검증자/교차 점검자 | 가설 제시 → 반증 요청 | 중간 |
| Level 3 | 공동 탐구자/확장 엔진 | 개방형 관찰 공유 → 개념 확장 | 높음 |

Structure Recognition Research Program은 Level 3 협업의 장기 사례 연구이다.

### 2.2 Level 1 — 도구/계산기 활용 실습 시나리오

**학습 목표:** AI를 계산·검색·코드 생성 도구로 효과적으로 활용하는 방법을 익힌다.

**시나리오: "4변수 Boolean 함수 중 단조 함수는 몇 개인가?"**

```
실습 단계:
1. 학생이 문제를 AI에게 직접 질문한다.
2. AI가 Dedekind 수 D(4) = 168을 제시하고, 전수 열거 방법을 설명한다.
3. AI가 파이썬 코드 초안을 제공한다 (is_monotone 함수).
4. 학생이 코드를 실행하여 168개를 직접 확인한다.

성찰 질문:
- AI의 답변을 왜 직접 코드로 검증했는가?
- AI가 틀릴 수 있는 경우는 어떤 경우인가?
```

**핵심 학습:** AI 출력물을 블랙박스로 신뢰하지 않고, 도구로서 사용하되 결과를 직접 검증하는 태도.

### 2.3 Level 2 — 검증자 활용 (가설 교차검증) 실습 시나리오

**학습 목표:** 자신의 이론적 주장을 AI를 이용해 다양한 각도에서 반증 시도를 해보고 가설을 강화한다.

**시나리오: "Space(XOR)의 S₄-궤도는 10개다"라는 주장 검증**

```
실습 단계:
1. 학생이 이론적 계산으로 10궤도 예측값을 도출한다 (Burnside 공식).
2. AI에게 "이 주장이 틀릴 수 있는 경우를 최대한 열거해달라"고 요청한다.
3. AI가 반례 후보들을 제시한다:
   - Burnside 적용 시 |S₄| = 24 가정이 맞는가?
   - 궤도 열거 알고리즘에 버그가 있는가?
   - Space(XOR)의 함수 개수가 32개가 맞는가?
4. 각 반례 후보를 코드로 직접 검증하고, 10개가 옳음을 확인한다.

성찰 질문:
- AI가 제시한 반례 중 실제로 문제가 된 항목이 있었는가?
- 이 과정이 "AI에게 답 얻기"와 어떻게 다른가?
```

**핵심 학습:** AI는 자신의 주장을 강화하는 도구가 아니라 비판적 검토 파트너로도 활용될 수 있다.

### 2.4 Level 3 — 질문 생성 및 공동 탐구자 실습 시나리오

**학습 목표:** 명확히 정의되지 않은 관찰에서 출발하여 AI와 함께 연구 가능한 질문을 생성하는 경험을 한다.

**시나리오: "K-map에서 체커보드 패턴이 눈에 띈다"**

```
실습 단계:
1. 학생이 체커보드 패턴에 대한 초기 관찰을 AI에게 공유한다.
   "4변수 K-map에서 이 패턴이 계속 눈에 띄는데, 왜 이렇게 특별해 보이는지 모르겠다."

2. AI가 여러 프레임워크를 제안한다:
   - 대칭성 프레임: 어떤 변수 순열에도 불변
   - 군론 프레임: S₄-불동점
   - 정보이론 프레임: 최대 엔트로피 함수
   - SRT 프레임: H3(주의 필터) 관점에서 설명

3. 학생이 자신에게 의미 있는 프레임을 선택하고, AI에게 그 프레임에서 생성되는
   후속 질문들을 제안해달라고 요청한다.

4. 학생이 제안된 후속 질문들 중 자신이 탐구하고 싶은 것을 선택한다.

5. 이 과정 자체를 메타 관찰의 대상으로 삼는다:
   "어떤 질문이 나에게 흥미롭게 느껴지는가, 왜인가?"

성찰 질문:
- AI가 제안한 프레임 중 연구자가 스스로 생각하지 못했을 것은 무엇인가?
- 어떤 기준으로 프레임을 선택했는가?
- 이 과정에서 연구자와 AI의 역할은 각각 무엇이었는가?
```

**핵심 학습:** AI는 가능한 설명 공간을 확장하는 "확장 엔진"이며, 연구자는 그 공간에서 의미 있는 방향을 선택하는 "선별자"이다. 이것이 H1·H7(구조 발견자 가설)의 Human-AI 협업 모델에 대응한다.

## 3. 기억 분담 모델과 교육 프레임워크의 연결

Level 1→3로 갈수록 협업에서 M_doc(t)의 역할이 변한다:

| Level | M_doc(t)의 역할 | AI의 기여 방식 |
|-------|--------------|-------------|
| Level 1 | 최소 (문제만 제공) | 단일 정답 또는 코드 제공 |
| Level 2 | 가설 명시적 포함 | 반례·약점 열거 |
| Level 3 | 관찰·미해결 질문 포함 | 프레임 확장·후속 질문 생성 |

이 프레임워크에서 **SESSION_START.md는 Level 3 협업을 위한 최적화된 M_doc(t) 구조**이다: 단순 답 요청이 아니라, 연구의 미해결 질문과 구조적 맥락을 담고 있어 AI가 확장 엔진으로 기능할 수 있게 한다.
