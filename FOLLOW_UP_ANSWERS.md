### FQ-1. Size of Space(G) for n=2,3,4,5 — Growth Rate Patterns?

**Question:** Systematically enumerate the size of Space(G) for single gate G at n=2,3,4,5. What growth rate pattern emerges as n increases?

**Currently Known:**
- For functionally complete gates (NAND, NOR): Space(G) = all 2^(2^n) functions. Growth is doubly exponential.
- For XOR (affine functions over GF(2)): |Space(XOR)| = 2^(n+1). For n=4, that is 2^5 = 32 affine functions. Growth is singly exponential.
- For AND (monotone subset): The size equals the number of antichains in the power set lattice of {1,...,n} (Dedekind numbers), which grows super-exponentially but much slower than doubly exponential.

| Gate | n=2 | n=3 | n=4 |
|------|-----|-----|-----|
| NAND/NOR | 16 | 256 | 65,536 |
| XOR | 4 | 8 | 32 |
| AND (monotone subclass) | 6 | 20 | 168 (Dedekind D(4)) |

**Connection to Existing Research:**
- Paper 2 (Repo 2) classified all 65,536 four-variable functions by layer structure. The 32 affine functions are a specific sub-population within those 65,536.
- Paper 3 (Repo 3) studied equivalence under variable rearrangement — the number of equivalence classes within Space(G) is a natural extension.

**Open Direction:**
- Map Space(G) for other common gates (OR, XNOR, NAND, Majority) across n=2,3,4.
- Visualize how the "density" of Space(G) within the full 65,536 changes by gate.

### FQ-2. Post Lattice Mapping of Space(G) — Which Level in the Hierarchy?

**Question:** Where does Space(G) sit in Post's lattice of clones? If visualized, which layer of the hierarchy does it occupy?

**Currently Known:**
- Post (1941) proved the lattice of Boolean clones is countably infinite but completely characterized.
- The five maximal clones are: T₀ (truth-preserving), T₁ (falsity-preserving), M (monotone), S (self-dual), L (affine/linear).
- Space(NAND) = Space(NOR) = full clone (top of lattice).
- Space(XOR) = L ∩ S = self-dual affine functions... actually Space(XOR) = the clone generated by XOR, which is the affine functions (L).
- Space(AND) is a sub-clone of M (monotone functions).

**Connection to Existing Research:**
- The Karnaugh map geometry of each Post class (FQ-6) directly connects to Paper 1's visual invariance work.
- SRT H2 (structural invariance) predicts that functions within the same Post class should share Karnaugh map characteristics.

**Open Direction:**
- Create a visual diagram: Post's lattice overlaid with Karnaugh map pattern examples for each major clone.
- This could become a standalone educational visualization (potential Repo 7: Post Lattice × Karnaugh Map Geometry).

### FQ-3. Intersection Space(G₁) ∩ Space(G₂) — Mathematical Structure?

**Question:** What is the intersection of two gate-generated spaces? Does it form a meaningful mathematical structure?

**Currently Known:**
- The intersection of two clones is always a clone (clones are closed under intersection).
- Space(XOR) ∩ Space(AND) = ? Both affine functions and monotone functions must hold. The only monotone affine Boolean functions are constants (0 and 1) and single-variable projections (x₁, x₂, ..., xₙ). This is a known result.
- More generally, intersections correspond to clones lower in Post's lattice.

**Connection to Existing Research:**
- The intersection problem is equivalent to asking: "which functions are constructible from both G₁ and G₂ individually?" — a constraint version of Q3.

### FQ-4. Closure Properties of Space(G) — Systematic Classification

**Question:** What are the closure properties of Space(G)? (Composition closure, negation closure, duality closure)

**Currently Known:**
- By definition, Space(G) is closed under composition (it is a clone).
- Negation closure: Space(NAND) is negation-closed (since NAND can simulate NOT). Space(AND) is not (AND is not negation-closed).
- Duality closure: a clone C is duality-closed if f ∈ C implies the dual f* ∈ C. Space(XOR) is duality-closed (XOR is self-dual in a generalized sense).

| Gate | Composition | Negation | Duality | Self-Dual |
|------|-------------|----------|---------|-----------|
| NAND | ✅ | ✅ | ✅ | — |
| NOR | ✅ | ✅ | ✅ | — |
| XOR | ✅ | ✅ | ✅ | ✅ |
| AND | ✅ | ❌ | ❌ | — |
| OR | ✅ | ❌ | ❌ | — |

**Connection to Existing Research:**
- These closure properties directly map to the structural invariants studied in Papers 1–3.
- Paper 2's checkerboard analysis implicitly uses duality: XOR and XNOR (its dual) form complementary checkerboard patterns.

**Open Direction:**
- Create a complete closure property table for all single-variable Boolean gates.
- Relate closure properties to visual invariants on Karnaugh maps.

### FQ-5. Minimal Generating Set Size vs. Space(G) Properties

**Question:** What is the relationship between the size of the minimal generating set and the properties of Space(G)?

**Currently Known:**
- NAND requires just 1 function to generate all Boolean functions (it is a basis by itself).
- {AND, NOT} is a basis of size 2.
- {AND, OR, NOT} is redundant (OR is definable from AND and NOT).
- Smaller generating sets tend to produce larger (more expressive) function spaces.

**Connection to Existing Research:**
- This connects to Q3's constructibility framework: the generating set size reflects "expressiveness."
- Paper 4's SRT H4 (research-worthiness) might predict: humans find minimal generating sets more "interesting" precisely because they encode more information in simpler form.

**Open Direction:**
- Formalize the relationship: is there a monotone correspondence between minimal generating set size and Space(G)?
- Investigate: does reducing generating set size always increase Space(G) size?

### FQ-6. Geometric Patterns of Linear/Monotone/Self-Dual Functions on Karnaugh Maps

**Question:** What geometric patterns do linear, monotone, and self-dual functions form on Karnaugh maps? What inclusion/exclusion relationships exist?

**Currently Known:**
- **Linear (affine) functions** on 4 variables: there are 32 (2^(4+1)). Their Karnaugh map patterns are checkerboard-like — formally, any affine function has the property that its "1-region" and "0-region" are balanced in a specific way. XOR is the canonical example (full checkerboard in Gray Code arrangement).
- **Monotone functions** on 4 variables: the 168 monotone functions (Dedekind number D(4)). Their Karnaugh maps have the property: no "isolated 1" is surrounded by 0s (moving from 0→1 is monotone in each variable). They form "upward-closed" sets in the Boolean lattice.
- **Self-dual functions**: f(x) = ¬f(¬x). Exactly half of all minterms are 1. Their Karnaugh maps have equal numbers of 0s and 1s, and the pattern is its own "complement under complement inputs."

**Connection to Existing Research:**
- Paper 1 (Repo 1): the checkerboard is precisely the Karnaugh map signature of XOR, the canonical linear function.
- Paper 2 (Repo 2): layer structure (Hamming weight layers) explains XOR's checkerboard through the odd-layer selection pattern.
- Paper 3 (Repo 3): variable rearrangement invariance — monotone functions are invariant under variable permutation but not necessarily monotone functions under all rearrangements.

**Open Direction:**
- Visualize all 32 affine functions on 4×4 Karnaugh maps — are there visual invariants beyond checkerboard?
- Visualize a sample of the 168 monotone functions — what is the visual "signature" of monotonicity?

### FQ-7. Visual Invariant for Space(G) — Do All Functions Share a Common Pattern?

**Question:** Do functions in Space(G) share a common visual invariant on Karnaugh maps?

**Currently Known:**
- For Space(XOR) = affine functions: **yes**, they share a "balanced pattern" structure. The checkerboard is the extreme case; others show balanced 8/8 splits with specific geometric structures.
- For Space(NAND) = all functions: trivially, no common visual invariant (all patterns possible).
- For Space(AND) = monotone: the visual invariant is "no isolated 1 cell surrounded by 0 cells" — a topological property of the pattern.

**Connection to Existing Research:**
- This is a direct application of SRT H2: "structural invariance under representation change." The representation changes from algebraic to visual (Karnaugh map), and the question is which properties are preserved.

**Open Direction:**
- Systematically search for visual invariants for Space(OR), Space(NAND with bounded depth), Space(Majority).
- Define formal "visual invariant" criteria that can be checked algorithmically.

### FQ-8. Is the Repo 1 Checkerboard Pattern the Geometric Expression of Space(XOR)?

**Question:** Is the checkerboard pattern studied in Paper 1 (Repo 1) the Karnaugh map geometric expression of Space(XOR)? What evidence does SRT H2 provide?

**Currently Known:**
- XOR on 4 variables = F(A,B,C,D) = A⊕B⊕C⊕D. This is an affine function (member of Space(XOR)).
- In Gray Code arrangement: XOR produces a perfect checkerboard (adjacent cells always differ → XOR output always flips between adjacent cells).
- Paper 1 (Repo 1) documented this checkerboard as a "structural invariance" — the pattern persists across equivalent Karnaugh map representations.
- SRT H2 states: "structural properties are preserved across representation transformations." The checkerboard is preserved when the Karnaugh map arrangement changes (from Gray Code to other arrangements, the checkerboard may become a double-diagonal, but the structure is still recognizable).

**Connection to Existing Research:**
- Paper 2 (Repo 2) showed: XOR selects odd Hamming weight layers (L₁ ∪ L₃). In Gray Code arrangement, adjacent cells differ by exactly 1 bit → adjacent layers alternate → checkerboard. This gives the mechanism behind Paper 1's observation.
- **Synthesis:** Paper 1 observed the phenomenon; Paper 2 explained the mechanism; Repo 6 places it in the algebraic framework (XOR generates the affine clone, and affine functions form "balanced" Karnaugh map patterns).

**Open Direction:**
- Does every member of Space(XOR) (not just the 4-variable XOR itself) show some form of "structural balance" on Karnaugh maps?
- Can we characterize the full class of Karnaugh map patterns that arise from affine functions?

### FQ-10. Gate Complexity and Space(G) Properties

**Question:** How does the gate complexity (minimum number of gates of type G needed to implement target function f) relate to the properties of Space(G)?

**Currently Known:**
- Gate complexity is only meaningful for functionally complete gates (NAND, NOR, etc.).
- For non-complete gates: functions not in Space(G) have infinite gate complexity (impossible to implement).
- For NAND: all 65,536 functions are implementable, but complexity varies. XOR requires at least 4 NAND gates.

**Connection to Existing Research:**
- Paper 4 (SRT) H4: "research-worthy structures are recognized by their informativeness given their complexity." Gate complexity is a natural measure of "complexity" for Boolean functions.
- Functions with low gate complexity but high visual "interestingness" may be prime examples for SRT analysis.

**Open Direction:**

### FQ-12. Mathematical Structure of "Functions Not Constructible from Gate G"

**Question:** What is the mathematical structure of {f : f ∉ Space(G)}? What is its meaning in clone theory?

**Currently Known:**
- The complement of a clone is not a clone (clones are defined by closure, complements are not closed).
- However, the complement has structure: it is the union of all clones that are NOT super-clones of the clone generated by G.
- In Post's lattice, everything below G's clone is constructible; everything above (if G's clone is not maximal) is not.

**Connection to Existing Research:**
- For Space(XOR) = affine functions: the 65,504 non-affine functions form the complement. They are characterized by having non-zero "non-linearity" — a quantity studied in cryptography.
- SRT H4: the non-constructible functions may be the "research-worthy" objects precisely because they escape the current framework.

**Open Direction:**
- Characterize the non-affine functions by their Karnaugh map patterns. What visual property distinguishes them from affine functions?
- Connect to the cryptographic concept of Boolean function non-linearity.

### FQ-18. Upper Limit of Finitely Classifiable Variable Count — Computational vs. Theoretical Limits

**Question:** Is there a variable count n beyond which classification becomes computationally infeasible? Where are the computational vs. theoretical limits?

**Currently Known:**
- Computational limits:
  - n=4: 65,536 functions — fully enumeratable
  - n=5: 4,294,967,296 ≈ 4.3 billion functions — large but feasible for some tasks
  - n=6: ~1.8 × 10^19 functions — beyond exhaustive enumeration
- Theoretical limits: none (Boolean function theory is well-defined for all n).
- The Dedekind numbers (counting monotone functions) are only known up to n=9 as of recent computations.

**Connection to Existing Research:**

**Open Direction:**

### FQ-19. Generalization to Ternary / Many-Valued Logic — Does the Same Framework Hold?

**Question:** Can the Boolean function space theory framework generalize to ternary or many-valued logic? Does the same theory hold?

**Currently Known:**
- In many-valued logic: instead of {0,1}, inputs and outputs range over {0,1,...,k-1} for some k ≥ 3.
- An n-variable k-valued function has k^(k^n) possible distinct functions — super-exponentially larger than Boolean.
- Post's theorem generalizes: the lattice of clones over k-valued logic is more complex.
- For k=3 (ternary), the lattice of clones is known to be uncountably infinite — fundamentally different from the Boolean case.

**Connection to Existing Research:**
- The Karnaugh map (a 2D grid) is specific to the 2-valued Boolean case. Generalization would require higher-dimensional or differently structured representation tools.
- SRT H2 (structural invariance) might apply: are there visual invariants in ternary Karnaugh-like structures?

**Open Direction:**
- This is a distant future direction. The Boolean case (n=4) should be fully developed first.
- Potential Repo 8: Ternary Function Space Theory (long-term future).

### FQ-20. Boolean Function Space as a Topological Space — Topological Properties of Space(G)?

**Question:** When viewed as a topological space, what are the topological properties of Space(G)? How does this correspond to Karnaugh map geometry?

**Currently Known:**
- The Boolean function space {0,1}^(2^n) is a finite set — trivially a discrete topological space with the discrete topology.
- More interesting: define a metric on Boolean function space. The Hamming distance between two functions (number of input points where they differ) defines a metric.
- Under this metric, Space(XOR) = the affine functions form a subspace. The "distance" between two affine functions equals the Hamming weight of their XOR (which is another affine function).
- Space(NAND) = the entire space (trivially connected, diameter = n).

**Connection to Existing Research:**
- The Hamming distance metric on function space connects to the Hamming weight layer structure in Paper 2 (Repo 2).
- On Karnaugh maps: Hamming distance between two functions equals the number of cells where they differ — a visual measure of "how different" two patterns look.

**Open Direction:**
- Define the "neighborhood structure" on Boolean function space: which functions are "close" to a given function?
- Investigate: are the "boundaries" of Space(G) (functions that are in Space(G) but barely so, by Hamming distance) related to structurally interesting functions?
- This connects to potential Repo 8: Boolean Function Space Topology.
